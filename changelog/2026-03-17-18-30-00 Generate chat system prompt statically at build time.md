# Generate chat system prompt statically at build time

## What changed

- `scripts/generate-changelog.mjs` now additionally generates `lib/generated/system-prompt.ts` — a TypeScript module that exports the full `SYSTEM_PROMPT` string with PRIMORDIA.md and the last 30 changelog filenames baked in at build time.
- `app/api/chat/route.ts` now simply imports `SYSTEM_PROMPT` from `@/lib/generated/system-prompt` instead of reading from the filesystem at runtime. The `fs`, `path`, and `loadPrimordiaContext` code have been removed.
- `lib/generated/` added to `.gitignore` (build artifact).
- `PRIMORDIA.md` updated to document the new `lib/generated/system-prompt.ts` artifact and clarify the build-time generation flow.

## Why

The previous implementation read `PRIMORDIA.md` and scanned `changelog/` via `fs` on every server cold-start. Using a static import instead means:
1. The prompt is bundled into the Next.js route at build time — no filesystem access at runtime.
2. It uses `changelog.json` (already generated by the prebuild step) as the authoritative source for changelog filenames, avoiding a second directory scan.
3. The last 30 entries are capped in the script, keeping the prompt a predictable size regardless of how many changelog files accumulate.
